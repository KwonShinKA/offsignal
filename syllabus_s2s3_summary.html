어떻게 작동하는가
내가 만든 에이전트를 타인이 써보는 순간 — 혼자서는 보이지 않던 것들이 드러납니다. 간극은 오류가 아니라 정보입니다. 타인의 반응이 드러내는 암묵지, 반복 사용이 파는 사고의 홈, 외재화된 사유의 귀환, 오류가 드러내는 숨겨진 전제들 — 검증의 현실을 철학적으로 읽는 10주입니다.

1시즌 수료자 / 에이전트 설계 워크숍(2시간·3만원) 수료 후 합류 가능 / 2시즌과 병행 운영 예정인 1시즌 2기 수강도 가능
Part 01 · 검증의 현실 — 타인의 손에서 드러나는 것들
W1
에이전트와 살아보기 — 나와 타인 사이의 간극
교차 사용 첫 경험. 간극의 세 유형: 의도와 다른 반응 / 예상 못한 상황 / 에이전트는 맞는데 내가 어색한 경우
워크시트: 첫 교차 사용 기록 — 어긋남의 유형 파악
W2
사용 패턴의 정치학 — 나는 무엇을 반복하고 있는가
반복 사용이 만드는 사고의 홈(groove). AI가 잘 답하는 방식으로만 질문을 재구성하는 무의식적 수렴
워크시트: 프롬프트 패턴 분류 — 확인형 / 생성형 / 판단 위임형
W3
종속과 도구화 사이 — 이분법을 넘어서
AI가 나를 확장시키는 영역과 내가 AI에 맞춰 수축하는 영역의 동시적 공진화(co-evolution)
워크시트: 사유 지형도 비교 — 1시즌 첫 주와 현재
Part 02 · 변형의 조건 — 기술이 사유를 어떻게 재구성하는가
W4
외재화된 사유의 귀환 — 아웃소싱한 생각은 어디로 가는가
기억·판단·감정의 외재화가 내부 구조에 미치는 영향. 스티글레르의 tertiary retention 경유
워크시트: AI 없이 못하는 것의 목록 — 의식적/무의식적 위임 구분
W5
프롬프트 언어와 사유의 범위 — 말하는 방식이 사고를 만든다
짧고 명확한 프롬프트의 효율성과 사유 단순화의 위험. 비트겐슈타인의 '언어 게임(language game)'
워크시트: 같은 질문, 5가지 언어 구조로 프롬프팅
W6
집단 에이전트와 공유된 사유 — 나의 설계가 타인의 맥락에서 작동할 때
암묵지(tacit knowledge) — 설계자가 말로 표현하지 못한 전제들. 3시즌 예고
워크시트: 내 에이전트를 타인에게 설명하기 — 설명 안 되는 지점이 암묵지
Part 03 · 재설계 — 설계자는 어떻게 자신을 갱신하는가
W7
에이전트 업데이트의 철학 — 무엇이 바뀌어야 하고 무엇이 지켜져야 하는가
기술 업데이트의 리듬 vs 내 철학 변화의 리듬. 업데이트 기준을 성능이 아닌 가치 변화에 두는 감각
워크시트: 7개 레이어 재검토 — 유지·수정·삭제 + 이유
W8
오류와 창의성 — 틀림은 어떻게 새로운 사유를 여는가
AI의 hallucination 재해석 — 오류는 나의 숨겨진 전제를 꺼내는 도구
워크시트: 가장 당황스러웠던 AI 응답 — 그 기대 뒤에 숨은 전제는?
W9
나의 철학적 에이전트 v2.0 — 재설계 결과 공유
레이어 업데이트 완성 및 공유. 핵심은 업데이트된 에이전트가 아니라 왜 바뀌었는가의 서사
공유: ① 가장 많이 바뀐 레이어 ② 바뀐 이유 ③ 바꾸지 않은 것
W10
설계자로 산다는 것 — 기술 시대의 지속가능한 주체성
설계는 한 번 하고 끝나는 게 아니라 살면서 계속 갱신하는 행위. 3시즌 예고

Season 03
사유의 생태계
날짜 미정 · 6주 · 온라인 소그룹
하나의 에이전트에
여러 사람의 사유를 담을 수 있는가
여러 사람이 함께 하나의 에이전트를 설계할 때 충돌은 피할 수 없습니다. A는 판단을 유보하는 에이전트를, B는 명확한 입장을, C는 감정 반응을, D는 논리 구조를 원할 때 — 타협이 되는가, 아니면 근본적으로 충돌하는가. 충돌을 해소하는 방법이 아니라, 충돌 자체를 설계 안에 넣는 것이 3시즌의 핵심입니다.
기간
6주
형식
온라인 소그룹
최소 인원
8명 이상
참여 조건
1·2시즌 수료자 대상 (3인 이상 팀, 최소 2팀 이상일 때 개강). 3시즌은 각자의 사유 설계도 v2.0이 팀 작업의 재료가 됩니다.
집단 설계의 세 가지 방향 — 충돌 해소 방식
방향 A
타협의 설계
충돌하는 가치들 사이 공통 분모 추출. 무엇을 공유하고 무엇을 포기할지 결정하는 과정이 에이전트의 철학이 됩니다.
방향 B
공존의 설계
복수의 관점이 긴장 상태로 공존하는 에이전트. 충돌 자체를 설계 안에 넣는 가장 급진적인 방향.
방향 C
역할 분리의 설계
단일이 아닌 역할별 복수 에이전트 앙상블. 누가 어떤 에이전트를 담당하는가의 구조 설계.
6주 일정
W1
사유의 충돌 — 나의 설계도를 테이블 위에 놓기
각자의 사유 설계도 v2.0 발표. 팀 구성 및 첫 핵심 질문 도출. 충돌 유형 식별
팀 작업: 공유·비공유 지도 — 충돌 지점 시각화
W2
협상의 구조 — 충돌을 어떻게 다룰 것인가
타협 / 우선순위 / 공존 / 역할 분리 중 어떤 방식으로 갈 것인가. 그 선택 자체가 에이전트의 철학
팀 작업: 충돌 처리 방식 결정 + 팀별 철학 명시화
W3
집단 설계도 작성 — 우리의 에이전트는 무엇을 구현하는가
7개 레이어를 팀 단위로 작성. 개인 설계도와 무엇이 달라졌는가. 미결 과제 식별
팀 작업: 집단 설계도 초안 완성
W4
에이전트 구현 및 교차 테스트 — 설계가 실제로 작동하는가
팀 간 교차 테스트 — 다른 팀의 에이전트를 사용해보기. 같은 질문에 두 팀이 어떻게 다르게 답하는가
팀 작업: 교차 테스트 보고 — 수정할 것 vs 수정하지 않을 것
W5
사회적 맥락으로 — 이 에이전트는 어떤 세계를 만드는가
알고리즘 편향, 데이터 권력, 집단 설계의 책임. 우리가 함께 설계한 것이 세계에 어떤 흔적을 남기는가
팀 작업: 사회적 시나리오 분석 — 어떤 편향이 내재하는가
W6
설계자들의 선언 — 우리는 어떤 방식으로 AI와 살 것인가
팀별 최종 발표. 집단 설계 경험 이후 개인의 사유 변화. 설계자 선언문 v3.0 갱신
마무리: 혼자일 때와 함께일 때 — 무엇이 흔들렸고 무엇이 단단해졌는가
3시즌 로드맵의 정신
에이전트는 완성되지 않는다.
설계는 삶의 방식이다.
1시즌: 나의 철학을 기계 안에 이식했다.
2시즌: 이식한 이후 내 설계가 타인에게 어떻게 다가가는지 물었다.
3시즌: 변화한 우리가 함께 무엇을 설계하는가를 묻는다.

기계의 속도에 소외되지 않고 — 사유의 주권을 되찾는 시간.
